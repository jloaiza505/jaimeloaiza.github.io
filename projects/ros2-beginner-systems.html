ROS2 Beginner System

Robotics & Control

Publisher, subscriber, service, and launch demo showcasing core ROS2 concepts.

Executive Summary

A portfolio-ready ROS2 project that demonstrates workspace setup, package structure, nodes, topics, services, and a launch file in one coherent system.

Objectives

Build a minimal ROS2 system that publishes a message, subscribes to it, and toggles publishing via a service call, all orchestrated through a launch file.

Role

Designed the system architecture, implemented the nodes and service interface, and documented build/run steps for reproducibility.

System / Methods

ROS2 workspace with separate talker, listener, and service packages plus a launch file that starts all nodes together.

Implementation

Implemented a 1 Hz talker on /chatter, a listener with timestamped logs, and a service that toggles publishing state. Added a launch file to run the full demo in one command.

Behavioral Requirements

Talker publishes only when enabled, the service toggles that state, and the listener always logs received messages. Documented build, source, and launch commands in the README.

Build & Validation

Verified messaging in terminal output, confirmed service toggling behavior, and tested the launch flow after colcon build and environment sourcing.

Results

End-to-end ROS2 demo with reliable topic communication, service control, and repeatable launch.

Challenges & Tradeoffs

Balanced simplicity for learning with enough structure to be portfolio-ready and reusable.

Next Steps

Add configurable publish rates, logging levels, a simulated sensor topic, and a simple action server.
